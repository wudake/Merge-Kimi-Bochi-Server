## 一、项目概述

### 1.1 背景
本工具面向中国门窗制造工厂的海外社媒运营团队，解决多平台、多账号、多人协作场景下的内容生产、发布、质量追踪及询盘转化管理问题。

### 1.2 目标
- 实现 TikTok / Instagram / YouTube / Facebook 四大平台的统一管理
- 建立从选题 → 生产 → 发布 → 效果追踪的管理闭环（无审批流程，数据手动录入）
- 以"有效询盘数"为核心指标，驱动运营决策
- 降低运营管理人员的管理成本，提升运营人员的执行效率

### 1.3 核心价值
- **管理人员**：一眼看清团队产出效率、账号健康度、询盘转化漏斗
- **运营人员**：清晰的任务清单、便捷的素材获取、直观的数据反馈

### 1.4 已确认关键决策（13项）

| # | 决策项 | 确认结果 |
|---|--------|----------|
| 1 | 多平台分发 | 仅记录关联账号，内容由运营手动到各平台发布，发布后回填链接。系统不调用任何平台 API |
| 2 | NAS 访问 | 点击路径自动打开 smb:// 协议链接；浏览器拦截时提供「复制路径」降级方案 |
| 3 | 报表导出 | Excel + PDF 两者都要 |
| 4 | 审批流程 | 不需要审批，运营专员自行维护状态流转 |
| 5 | 部署环境 | 本地服务器 / 内网部署，不暴露公网 |
| 6 | 域名证书 | 直接使用 IP 地址访问，暂不配置 HTTPS |
| 7 | 数据库备份 | 每天增量备份 + 每周全量备份，保留最近 30 天 |
| 8 | 账号体系 | 系统独立管理用户账号，不需要 SSO 对接 |
| 9 | UI 设计 | 无设计师，直接使用 Ant Design 5.x 默认风格 |
| 10 | 图表风格 | 简洁数字卡片 + 折线图/柱状图为主 |
| 11 | 平板适配 | 低优先级，暂不处理，MVP 仅支持桌面浏览器 1920x1080 |
| 12 | 历史数据导入 | 需要导入账号和选题库，内容和询盘从零开始 |
| 13 | 数据现状 | 历史账号和选题数据分散在多个 Excel 中，格式不统一 |

## 二、用户角色与权限体系

| 角色 | 职责 | 权限范围 |
|------|------|----------|
| **超级管理员** | 系统配置、账号分配、人员管理 | 全部功能 |
| **运营主管** | 团队管理、数据分析、策略制定 | 全部账号查看、导出报表、管理选题库、人员管理 |
| **运营专员** | 内容生产、账号日常运营 | 仅查看/操作被分配的账号、提交内容、查看个人数据 |

### 2.1 账号分配机制
- 每个运营专员可被分配 **多个平台账号**
- 支持账号按市场/品牌/产品线 **自定义分组**（如：主账号组、混剪账号组、备用账号组）
- 账号交接：人员变动时支持一键转移账号及历史数据归属

#### 3.1.3 分组展示方式（前端自选）

- 按运营人分组

#### 3.1.4 账号批量导入
- 支持历史账号数据从 Excel 批量导入
- 兼容多来源 Excel 格式，支持字段映射
- 导入错误行可导出查看

### 模块二：内容产出管理板块

#### 3.2.1 周度内容计划（核心视图）
- **周视图日历**：以周为单位展示内容排期
- 每天卡片展示内容条目，包含：内容标题、负责人、内容类型、社媒平台标签
- 支持列表视图切换
- 支持周切换（上一周/下一周/本周）

#### 3.2.2 内容字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 内容标题 | 文本（必填） | 内容主题名称 |
| 关联账号 | 多选 | 要发布的社媒账号（接入真实账号列表） |
| 内容类型 | 可输入 | 原创 / 混剪 / 搬运，支持输入自定义类型 |
| 内容链接（URL） | 文本 | 发布后回填的内容链接 |
| 内容脚本 | 长文本 | 视频脚本或内容大纲 |
| 内容文案 | 长文本 | 发布文案、标题描述等 |
| 负责人 | 单选用户 | 接入真实用户列表 |
| 发布时间 | 日期时间 | 计划发布时间 |
| 备注 | 长文本 | 其他补充信息 |

#### 3.2.3 内容状态流转（无审批）
```
待制作 → 制作中 → 待发布 → 已发布 → 已归档
```
- 运营专员自行维护状态，无审批环节
- 列表视图支持直接下拉修改状态
- 状态变更时自动更新关联选题的使用记录

#### 3.2.4 批量操作
- 批量创建内容任务
- 批量指派负责人
- 批量调整发布时间

#### 3.3.1 选题管理

#### 3.3.3 选题分类与检索
- 分类维度：产品类型、内容类型、适用场景
- 多维度筛选 + 关键词搜索
- 热门选题推荐（基于历史使用效果排序）

### 模块四：内容数据与报表板块（阶段二，待开发）

#### 3.4.1 数据指标追踪
针对每条已发布内容，追踪以下指标（**全部手动录入**）：

| 指标类别 | 具体指标 |
|----------|----------|
| 曝光互动 | 播放量/展示量、点赞数、评论数、分享数、收藏数、完播率 |
| 账号增长 | 粉丝净增长数、粉丝总量 |
| 询盘转化 | 私信询盘数、评论询盘数、主页点击数、带来的有效询盘数 |

#### 3.4.2 周度/月度产出报告
- 自动生成周报：本周发布内容数量、各平台分布、总播放量、总询盘数
- 自动生成月报：环比趋势、最佳内容 TOP5、待优化账号预警
- 支持按运营人员、账号、平台、时间段筛选导出
- 导出格式：Excel + PDF

#### 3.4.3 内容效果评级
- 自定义评级规则（如：播放量>1万且询盘>5 → S级）
- 自动评级 + 人工调整
- 高评级内容自动归档至"爆款素材参考"

#### 3.4.4 数据看板（管理人员首页）
- **今日/本周概览**：待发布任务数、本周已发布数、本周询盘总数
- **趋势图表**：近30天播放量趋势、近30天询盘趋势、粉丝增长曲线
- **排行榜**：本周最佳内容 TOP5、本月最佳运营人员、各账号健康度对比
- **异常预警**：连续7天无更新的账号、连续3条内容0询盘的账号

### 模块五：素材库管理板块（阶段三，待开发）

门窗工厂拥有大量产品视频、安装现场、工厂实拍等素材，需统一管理：

- **素材引用**：素材库不存储实际文件，仅记录素材的 NAS 路径、文件名称、缩略图（可选）、描述信息
- **素材分类**：产品素材、工厂实拍、安装案例、客户反馈、通用模板（片头片尾）
- **素材标签**：支持多标签（如：断桥铝、白色、卧室场景、2026春季）
- **素材检索**：按分类、标签、文件名搜索
- **素材复用统计**：统计每个素材被使用了多少次，用于哪些内容
- **素材权限**：部分敏感素材（如未上市新品）可设置查看权限
- **NAS 访问**：点击路径自动打开 smb:// 链接，浏览器拦截时提供「复制路径」按钮

### 模块六：询盘管理板块（阶段三，待开发）

B2B工厂的核心目标是获取有效询盘，需建立询盘跟踪闭环：

- **询盘录入**：由运营人员自行录入询盘信息（来源平台、来源内容、询盘时间、客户信息、需求描述），销售团队确认有效后运营在系统中标记为"有效询盘"
- **询盘状态**：待跟进 / 跟进中 / 已报价 / 已成交 / 已流失
- **询盘归属**：自动关联到带来询盘的内容和运营人员
- **转化率看板**：内容曝光 → 询盘 → 成交 的漏斗分析
- **客户池**：基础的客户信息管理（独立维护，暂不对接外部系统）

## 四、技术架构

### 4.1 架构选型
前后端分离 + 单体服务，内网部署。

```
客户端 (React 19 + Vite 6)
    ↓
Nginx (静态资源托管 + 反向代理)
    ↓
后端服务 (NestJS 11 + Node.js 22)
    ↓
PostgreSQL 16 (主数据库) + Redis 7 (缓存/会话)
```

### 4.2 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 前端框架 | React + TypeScript | 19.0.0 |
| 前端构建 | Vite | 6.4.2 |
| UI 组件库 | Ant Design | 5.24.x |
| 状态管理 | Zustand | 5.0.x |
| 路由 | React Router | 7.0.x |
| 后端框架 | NestJS | 11.0.x |
| ORM | Prisma | 6.19.3 |
| 数据库 | PostgreSQL | 16-alpine |
| 缓存 | Redis | 7-alpine |
| 认证 | JWT (passport-jwt) | 4.0.x |
| Excel/PDF | exceljs / pdfkit | 4.4.x / 0.16.x |

### 4.3 部署环境
- 本地服务器 / 内网 HTTP 部署
- 无域名、无 HTTPS 证书
- 系统仅在工厂内网访问

## 八、非功能性需求

### 8.1 性能要求
- 页面加载时间 < 2秒
- 支持同时在线用户：50人
- 素材文件不上传至服务器，仅记录 NAS 路径

### 8.2 安全要求
- 用户权限严格隔离，运营专员只能查看自己被分配的账号
- 敏感数据（客户信息）加密存储
- 操作日志留痕，支持审计

### 8.3 易用性要求
- 界面语言：**仅中文**
- 响应式设计：低优先级，暂不处理，MVP 仅支持桌面浏览器
- 素材引用：大文件存储在本地 NAS，系统中仅保存 NAS 访问路径

### 8.4 技术环境
- **NAS 访问协议**：Samba（SMB/CIFS），点击 smb:// 链接唤起本地文件管理器
- 周视图日历为核心交互

### 8.5 扩展性
